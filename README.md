# 2. 서울 상권 추천 AI 서비스

## 📖 Project Overview

서울시 공공데이터를 기반으로 행정구/동 단위 업종별 창업 추천 결과를 제공하는 AI 웹 서비스입니다.  
상권 데이터를 분석하고 머신러닝 모델을 학습하여 추천/보류/비추천 결과를 제공하도록 구현했습니다.

## 🗓 개발 기간

- 2025.11 ~ 2025.12

## 👥 Team

- 팀 프로젝트

## 🎯 핵심 기능

- 서울시 상권 데이터 분석
- 업종별 창업 추천 예측
- 머신러닝 모델 기반 추천 결과 제공
- 결과 리포트 제공
- 정보 공유 게시판 구현

## 🛠 기술 스택

- Python
- FastAPI
- React + Vite
- MySQL
- Docker / Docker Compose
- LightGBM
- CatBoost
- Pandas / NumPy

## 🙋 담당 역할

- 분석용 데이터 수집
- 데이터 전처리 및 EDA
- 분류 모델링 학습 및 분석
- 게시판 프론트/백엔드 구현
- 데이터 학습 모델 검증

## 🚀 주요 구현 내용

### 1) 데이터 전처리 및 EDA

- `dong_cd`, `business_cd`, `yqc_cd` 기준 데이터 정렬
- `pct_change()`를 활용한 성장률 계산
- 최근 3분기 기반 파생 변수 생성
  - `qs_roll3`
  - `qs_vol`
  - `qs_diff`
- `np.log1p()`를 활용한 로그 변환으로 이상치 영향 완화
- 업종별 성장률 정규화 수행

### 2) 머신러닝 모델링

- 추천 / 보통 / 비추천 3분류 타깃 정의
- LightGBM / CatBoost 모델 비교
- `train_test_split(..., stratify=y)`를 활용한 클래스 불균형 대응
- Validation 성능 기준으로 하이퍼파라미터 튜닝
- Feature Importance를 통한 핵심 피처 분석

### 3) 게시판 기능 구현

- 게시글 목록 조회
- 게시글 상세 조회
- 게시글 작성
- 게시글 수정
- 게시글 삭제
- 비밀번호 검증 기반 게시글 수정/삭제 처리

### 4) 백엔드 API 구현

- FastAPI 기반 `/board` 라우터 분리
- RESTful API 설계
- 상태 코드별 예외 처리
  - 400 Bad Request
  - 403 Forbidden
  - 500 Internal Server Error

### 5) 프론트엔드 구현

- React 기반 게시판 목록/상세/작성/수정 페이지 구현
- Pagination 적용
- axios 기반 API 연동
- 로딩 / 에러 처리

## 💡 Problem Solving

### 문제 1. 데이터 스케일 차이와 이상치

매출, 유동인구, 아파트 가격 등의 값 범위 차이가 커서 모델 학습이 불안정한 문제가 있었습니다.  
이를 해결하기 위해 `np.log1p()` 로그 변환을 적용하여 극단값 영향을 완화하고 학습 안정성을 높였습니다.

### 문제 2. 단순 현재 값만으로는 추세 반영 한계

한 시점의 데이터만으로는 상권의 성장 흐름을 설명하기 어려웠습니다.  
이를 해결하기 위해 최근 3분기 평균, 변동성, 증감량 등의 파생 변수를 추가해 시계열 패턴을 반영했습니다.

### 문제 3. 클래스 불균형

추천/보통/비추천 라벨 분포가 균일하지 않아 평가 왜곡 가능성이 있었습니다.  
`stratify`를 활용해 학습/검증 데이터 분포를 유지하도록 구성했습니다.

## ✅ 성과

- 데이터 분석부터 모델 학습, API 연결, UI 구현까지 전체 흐름 경험
- AI 모델을 실제 웹 서비스 형태로 연결
- 게시판 CRUD 기능까지 포함한 서비스 완성
- 데이터 기반 문제 해결 능력 강화

## 🔗 핵심 포인트

> 데이터 분석 결과를 모델로 연결하고,  
> 그 결과를 실제 사용자에게 제공하는 서비스 형태까지 구현한 프로젝트입니다.

# 3. Strawberry AI - 취업 준비 AI 에이전트

## 📖 Project Overview

sLLM과 RAG를 활용하여 취업 준비 과정을 지원하는 AI 에이전트 서비스입니다.  
서류 분석, 자기소개서 피드백, 예상 면접 질문 생성, 맞춤 채용공고 추천, AI 면접 챗봇 기능을 제공합니다.

## 👥 Team

## 🎯 핵심 기능

- 서류 분석

- 예상 면접 질문 생성
- 맞춤 채용공고 추천
- AI 면접 챗봇

## 🛠 기술 스택

- FastAPI
- React + Vite
- Node.js
- MySQL
- Redis
- Docker Compose

## 🙋 담당 역할

- 지피지기 면접 예상 질문 데이터 구축
- 면접 질문 분석 및 피드백 로직 고도화
- 사용자 답변 요약 및 피드백 기능 구현

## 🚀 주요 구현 내용

- 취업 준비 과정에서 필요한 핵심 기능을 AI 서비스로 통합
- 사용자 자기소개서와 직무 정보를 기반으로 예상 질문 생성
- 답변 내용을 분석하여 피드백 제공
- 취업 준비 흐름에 맞는 기능 구조 설계

## 💡 Problem Solving

- 취업 준비 과정은 정보 탐색, 자기소개서 작성, 면접 대비가 각각 분리되어 있어 사용자 부담이 컸습니다.
- 이를 해결하기 위해 여러 기능을 하나의 흐름 안에 배치하여, 사용자가 준비 단계를 끊기지 않고 이어갈 수 있도록 구성했습니다.
- 단순 챗봇이 아니라 취업 준비 상황에 맞는 기능 중심 AI 서비스로 확장한 점이 핵심입니다.

## ✅ 성과

- 최신 AI 서비스 흐름인 RAG/LLM 기반 프로젝트 경험
- 단순 CRUD가 아닌 실질적 사용자 문제 해결형 서비스 기획/구현
- 취업 도메인에 맞는 AI 기능 구조 이해

## 🔗 핵심 포인트

> 최신 AI 기술을 활용해  
> 사용자의 취업 준비 과정을 실질적으로 돕는 서비스로 구현한 프로젝트입니다.

## 📚 What I Learned

### 1. 데이터와 서비스는 연결되어야 한다

데이터 분석 결과가 좋더라도, 사용자가 실제로 활용할 수 있는 UI/API 형태로 연결되지 않으면 서비스 가치가 낮아진다는 점을 배웠습니다.

### 2. 백엔드는 기능 구현보다 구조 설계가 중요하다

CRUD 구현 자체보다, API를 어떻게 나누고 예외를 어떻게 처리할지 구조적으로 생각하는 습관이 중요하다는 점을 체감했습니다.

### 3. 협업은 개발 속도보다 방향을 맞추는 일이다

GitHub, Slack, Notion을 활용하며 팀원 간 역할 분담과 진행 상황 공유가 프로젝트 완성도에 큰 영향을 준다는 점을 배웠습니다.
